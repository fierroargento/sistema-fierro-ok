Ayuda APB - Sistema Fierro

Version actualizada al punto actual del sistema.

Esta ayuda explica como trabajar cada pedido sin improvisar, siguiendo estados, acciones sugeridas, alertas y mensajes prearmados.

1. Regla general APB

El sistema es el tablero operativo principal. Si el sistema muestra una accion sugerida, esa es la proxima accion correcta.

No saltear estados. Cada estado existe para dejar registro, evitar olvidos y ordenar responsabilidades.

No inventar mensajes, links ni pasos manuales. Usar siempre botones del sistema: WhatsApp, seguimiento, imprimir, reclamos, devoluciones y cierre.

No despachar si hay alerta critica. Si aparece reclamo activo, cancelacion, bloqueo o falta de datos, primero se resuelve eso.

2. Roles del sistema

El sistema tiene tres roles principales. Cada uno ve y puede hacer cosas distintas.

Carga

Responsable de completar datos, contactar clientes, controlar seguimiento post-despacho, gestionar reclamos, devoluciones y cierre final.

Debe mirar principalmente: Cargando Pedido, Despachado, Verificar llegada a destino, Listo para retirar, No entregado, Reclamos y Entregado pendiente de cierre.

Despacho

Responsable de imprimir etiquetas, preparar, embalar y despachar pedidos.

Debe mirar principalmente: Etiqueta Lista, Etiqueta Impresa y Embalado.

Regla APB: Despacho no inventa datos de seguimiento ni resuelve reclamos. Si falta algo, vuelve a Carga o Admin.

Admin

Puede operar todos los pasos, corregir datos, revisar integraciones, auditoria, usuarios y productos.

3. Tablero principal

El tablero muestra pedidos activos ordenados por prioridad. No es para mirar todo: es para trabajar lo urgente segun el rol.

Semaforo

Verde: dentro de tiempo normal.

Amarillo: requiere atencion.

Rojo: prioridad operativa. Revisar antes que el resto.

Accion sugerida

Es el paso concreto que hay que hacer ahora. Seguila siempre. Si el pedido no tiene accion clara, entrar al detalle y revisar el motivo.

Alertas del panel

Aparecen automaticamente cuando hay situaciones que requieren atencion:

  • Ventas de Mercado Envios sin etiqueta que no entraron a Fierro.
  • Pedidos sin despacho hace mas de 24 horas.
  • Pedidos Via Cargo para seguimiento logistico pasadas las 72 horas.
  • Pedidos Andreani que requieren revision de tracking.
  • Reclamos sin revisar hace mas de 24 horas.

4. Estados del pedido - flujo completo

1) Cargando Pedido

Que significa: el pedido todavia necesita revision o datos antes de pasar a preparacion.

Acciones posibles: completar carga, contactar cliente, enviar mensaje ML o imprimir etiqueta si el flujo ya lo permite.

Regla APB: no avanzar si faltan datos obligatorios. El sistema debe llevar al primer paso pendiente.

2) Etiqueta Lista

Que significa: el pedido ya tiene los datos necesarios y la etiqueta esta disponible para imprimir.

Accion: imprimir etiqueta.

Regla APB: imprimir debe abrir el flujo de impresion sin cerrar el trabajo del operador.

3) Etiqueta Impresa

Que significa: la etiqueta ya fue impresa y el pedido pasa a preparacion fisica.

Accion: marcar embalado.

4) Embalado

Que significa: el pedido ya esta preparado y listo para salir.

Accion: marcar despachado.

Regla APB: al marcar embalado o despachado, el operador debe quedar dentro del detalle para continuar el proceso.

5) Despachado

Que significa: el pedido ya salio fisicamente del deposito.

Acciones: avisar cliente despacho, controlar seguimiento, cargar reclamo si corresponde o avanzar cuando el transporte confirme movimiento.

Regla APB para Acordas la Entrega: el aviso de despacho no es opcional. Siempre se debe avisar al cliente con numero de seguimiento y link dinamico del transporte.

6) Verificar llegada a destino

Que significa: el pedido llego a destino o sucursal, pero todavia no se confirmo la entrega al cliente.

Accion obligatoria: si es entrega en sucursal, avisar al cliente por WhatsApp que ya puede retirar.

Mensaje: informa sucursal, seguimiento y recordatorio de 5 dias para retirar antes de que vuelva a origen.

Regla APB: no marcar entregado si antes no se aviso al cliente cuando corresponde retiro en sucursal.

7) Listo para retirar

Que significa: el cliente ya fue avisado de que puede retirar el pedido en sucursal.

Accion: marcar entregado cuando el cliente confirme retiro o se verifique la entrega.

Regla APB: el cliente tiene 5 dias para retirar antes de que el pedido pueda volver a origen.

8) Entregado

Que significa: el cliente recibio o retiro el pedido.

Acciones: enviar mensaje postventa y, si es Mercado Libre Acordas la Entrega, usar "Ya avise Mercado Libre" cuando corresponda.

Regla APB: en parrillas se envia mensaje especial con tips de uso y cuidado. En el resto de productos se envia mensaje general de postventa.

9) Finalizado

Que significa: el pedido ya no requiere accion operativa.

5. Estados especiales

Con demora de entrega

Que significa: el pedido supero tiempos esperados o requiere control de demora.

Acciones: revisar tracking, contactar transporte o cliente y cargar reclamo de envio si corresponde.

Con reclamo en transporte

Que significa: ya existe reclamo de envio cargado.

Acciones: revisar evolucion, actualizar observaciones y marcar revisado cuando corresponda.

Regla APB: si pasan mas de 24 horas sin revision, debe aparecer como prioridad.

No entregado

Que significa: la entrega fallo, el paquete vuelve o no llego al cliente.

Accion: gestionar devolucion.

Regla APB: no cerrar directo. Primero registrar devolucion, estado de productos y resolver reclamo si aplica.

Reclamar a Mercado Libre

Que significa: hay una devolucion o problema que requiere gestion especifica con Mercado Libre.

Accion: gestionar reclamo Meli.

Regla APB: si hay producto dañado, no cerrar como finalizado hasta registrar resultado del reclamo.

6. Flujo por canal

Mercado Libre - Mercado Envios

Se trabaja con la logica propia de etiqueta y seguimiento de Mercado Libre.

Regla APB: si hay reclamo activo en ML, aparece alerta de NO DESPACHAR. Primero se atiende el reclamo.

Seguimiento: ML lo gestiona internamente. No inventar datos.

Mercado Libre - Acordas la Entrega - Domicilio

El sistema requiere completar datos y seguimiento segun transporte.

Despachado: se debe avisar al cliente con seguimiento y link dinamico.

Entregado: al confirmar entrega, si corresponde, usar "Ya avise Mercado Libre".

Mercado Libre - Acordas la Entrega - Sucursal

Flujo operativo despues de despachado:

Despachado -> Verificar llegada a destino -> Listo para retirar -> Entregado -> Finalizado.

Regla APB: cuando llega a sucursal, avisar al cliente con sucursal, seguimiento y advertencia de 5 dias.

Cierre: despues de entregado, si corresponde, usar "Ya avise Mercado Libre".

Tienda Nube

El sistema sincroniza pedidos por webhook y permite re-sync desde el detalle cuando hace falta.

Link TN: desde el detalle se puede abrir la venta en Tienda Nube.

Tracking TN: usar el seguimiento o link que el sistema muestra en el resumen. No inventar links manuales.

Regla APB: si el pedido de TN aparece cancelado o bloqueado, no despachar.

Mayorista / Manual

Se carga manualmente desde Nuevo Pedido. Requiere datos completos de cliente, venta, entrega y productos.

Regla APB: si es Via Cargo, el seguimiento puede cargarse despues del despacho.

7. Transportes

Andreani

El sistema identifica pedidos Andreani y muestra bloque de estado Andreani.

Si no hay credenciales: debe quedar como Andreani sin credenciales configuradas y no romper la operacion.

Cuando haya credenciales: se podra actualizar estado Andreani desde el detalle.

Correo Argentino

Requiere etiqueta y seguimiento cuando el flujo lo pide.

Usar siempre el link de tracking que arma el sistema.

Via Cargo

Puede no tener seguimiento antes del despacho. El seguimiento suele aparecer despues.

Regla APB: no exigir seguimiento antes de despachar si el flujo real de Via Cargo no lo permite.

8. WhatsApp - uso obligatorio

Los mensajes de WhatsApp no son opcionales. Son parte del proceso operativo.

No escribir mensajes manuales. Usar siempre los botones del sistema para evitar errores, omisiones o textos distintos entre operadores.

Si el cliente no tiene telefono cargado, los botones de WhatsApp no aparecen. En ese caso, gestionar el contacto por ML, TN o el canal que corresponda.

Contacto inicial

Se usa para abrir conversacion por WhatsApp cuando ya corresponde continuar fuera de ML.

Hola! Soy Ezequiel de Fierro 100% Argento. Gracias por tu compra. Te escribo por aca para avanzar con el despacho.

Aviso de despacho

Obligatorio para Acordas la Entrega.

Incluye numero de seguimiento y link dinamico del transporte armado por el sistema.

Aviso de retiro disponible

Se usa en Verificar llegada a destino / sucursal.

Incluye sucursal, seguimiento y aviso de 5 dias antes de volver a origen.

Postventa

Parrillas: mensaje especial con tips de cuidado, limpieza y curado.

Otros productos: mensaje general de agradecimiento, ayuda y enlace a Instagram.

Regla APB: los mensajes operativos no usan emojis para evitar problemas de compatibilidad en WhatsApp Web.

9. Tracking y seguimiento logistico

El sistema abre automaticamente el seguimiento segun el transportista.

No hace falta buscar links manualmente ni copiar datos. Usar siempre el boton de seguimiento del pedido.

Andreani: abre la pagina de tracking con el numero cargado. Cuando haya credenciales, tambien podra consultar estado por API.

Via Cargo: abre la pagina de tracking con el numero cargado. El seguimiento puede aparecer despues del despacho.

Mercado Envios / Correo Argentino: usar el link armado por el sistema y el numero cargado en el pedido.

Tienda Nube: si TN provee URL de tracking, el sistema debe usar esa URL.

10. Reclamos y devoluciones

Reclamos de envio: se cargan desde el detalle del pedido en estados posteriores al despacho.

Revisado hoy: usarlo cuando se hizo seguimiento real del reclamo.

No entregado: usar Gestionar devolucion. Registrar que volvio, en que estado volvio cada producto y si corresponde reclamo con Mercado Libre.

Mercado Libre con producto dañado: no cerrar como finalizado hasta gestionar el reclamo Meli.

11. Administracion - solo Admin

Usuarios

Desde Admin -> Usuarios se pueden crear, editar, activar o desactivar usuarios.

Roles disponibles: admin, carga y despacho.

Productos

Desde Admin -> Productos se actualiza el padron subiendo un Excel (.xlsx):

  • Columna A: SKU.
  • Columna B: descripcion.

La carga reemplaza el padron actual. Verificar el archivo antes de subirlo.

Auditoria

Desde Admin -> Auditoria se ve el registro de acciones: usuario, pedido, accion, IP y fecha.

Integraciones

Mercado Libre: conectar o desconectar cuenta, sincronizar ventas y revisar estado de token.

Tienda Nube: sincronizar pedidos, registrar webhooks y testear conexion.

Regla APB: los webhooks se registran una sola vez o cuando cambia la URL del servidor.

12. Checklist rapido por operador

Carga

  1. Revisar pedidos con datos pendientes.
  2. Contactar cliente si el sistema lo indica.
  3. Controlar despachados, llegadas a destino, reclamos y no entregados.
  4. En Acordas, avisar ML cuando corresponda.
  5. No cerrar pedidos si falta reclamo, devolucion o confirmacion.

Despacho

  1. Imprimir etiqueta.
  2. Embalar.
  3. Despachar.
  4. No resolver reclamos.
  5. No inventar datos de seguimiento.

Admin

  1. Revisar integraciones.
  2. Revisar auditoria si hay dudas.
  3. Corregir pedidos bloqueados o mal cargados.
  4. Mantener productos y usuarios actualizados.
